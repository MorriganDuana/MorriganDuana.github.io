Push Jekyll on Github

How to do

Your site is finally finished but you try to put your jekyll site on github and it doesn't work for some obscure reason. Don't worry, I'll explain you how it works.

Preconfiguration

Start by creating a repo github withoutreadme. If you want a github page, you just have to call your repo by your nickname_github.github.io when you create it.

On the other hand, you will see the need to configure the "base url" parameter of your file config.yml located at the root of your jekyll site. It only remains to build jekyll with the order: 

jekyll build

Anonimization

The following will take place directly on your Linux terminal. Github has changed some of these policies. Until now your email address was in the logs linked to your repo. We're going to have to make some changes and then it's just as much for our privacy that our personal information is not out there in an uncontrolled way.

In the settings of your github, you have the email tab. You have a keep my email addresses private setting and below it, you have an email address that ends with users.noreply.github.com. Copy it and then change your github configuration: 

  git config user.email "New_email"

You can check if your email address is in the logs visibly with the command git log then you can fix your github history with the following script found here: 

#!/bin/sh

git filter-branch --env-filter '

OLD_EMAIL="your-old-email@example.com"
CORRECT_NAME="Your Correct Name"
CORRECT_EMAIL="your-correct-email@example.com"

if [ "$GIT_COMMITTER_EMAIL" = "$OLD_EMAIL" ]
then
    export GIT_COMMITTER_NAME="$CORRECT_NAME"
    export GIT_COMMITTER_EMAIL="$CORRECT_EMAIL"
fi
if [ "$GIT_AUTHOR_EMAIL" = "$OLD_EMAIL" ]
then
    export GIT_AUTHOR_NAME="$CORRECT_NAME"
    export GIT_AUTHOR_EMAIL="$CORRECT_EMAIL"
fi
' --tag-name-filter cat -- --branches --tags

Then push all the changes:

git push --force --tags origin 'refs/heads/*'

Push Jekyll on github

We will add our Jekyll on our github thanks to our configuration. To do this, we put ourselves in our folder created after the build. Then we are going to begin the commands git. 

#initialisation
git init

#On bascule sur une uatre branches
git checkout -b gh-Pages

#indexe les fichiers et pr√©pare au commit
git add .

#commit
git commit -m "First commit"

#synchornise
git remote add origin "adresse du github"

#push website on Github
git push origin gh-pages

I hope everything turned out well for you. In any case, you're supposed to have gotten through this tutorial without a hitch.
